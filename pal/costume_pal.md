# Costume PAL

## 定義

Costume PAL とは、同じ人物に付随する衣装・小物・着用状態を、
人物同一性やシーケンスの意味を壊さないように維持するために、
衣装の核・小物セット・状態差分・禁止再解釈を固定する継続層である。

Costume PAL は、単なる服装指定ではない。
見た目の飾りでもない。
Costume PAL は、衣装を

- 何を着ているか
- どの形で着ているか
- どの小物が含まれるか
- 何を変えてよく、何を変えてはならないか

として固定し、人物同一性とシーケンスの意味を
支えるための記述層である。

-----

## 目的

Costume PAL の目的は、服を綺麗に描かせることではない。
目的は、衣装が変化しても以下の状態を維持することである。

- 同じ衣装として読める
- 同じ場面の連続として読める
- 人物同一性を壊さない
- archetype drift を誘発しない

Costume PAL は、衣装を豪華にするためではなく、
人物とシーケンスの連続性を支えるために存在する。

-----

## なぜ Costume PAL が必要か

衣装は人物より二次的に見えるが、
実際には drift の大きな入口になる。

例：
- 花嫁化で欧米花嫁 archetype に流れる
- フォーマル化で generic glamorous woman になる
- 衣装の豪華化で人物が別類型に変わる
- 小物の消失で別シーンに見える

特にウェディングドレスのように、
文化的・儀式的な意味を強く持つ衣装は、
人物同一性より役割類型を先に呼び込むことがある。

したがって、衣装も背景と同様に
同一存在として管理される必要がある。

-----

## Costume PAL が固定するもの

### 1. Garment Core

衣装の最小同一性。その衣装をその衣装たらしめる骨格。

例：
- 白いマーメイドラインのウェディングドレス
- 袖なし
- オフショルダーまたはストラップレスの範囲
- ベールなし
- グローブなし

Garment Core は最小限に絞る。
細部を増やしすぎると、逆に別衣装へ再解釈されやすい。

### 2. Accessory Set

衣装に付属する小物や装身具。

例：
- 丸い細フレーム眼鏡
- 小さな上品なイヤリング
- 細いチェーンネックレス
- 結婚指輪
- ブーケ

Accessory Set は人物同一性とも結びつくため、
衣装の一部であると同時に人物の手掛かりでもある。

### 3. State of Wear

同じ衣装でも、どういう状態で身につけているかを固定する。

例：
- ブーケを持っている / 持っていない
- 指輪がはまっている / まだ目立たない
- ベールなしを維持
- 胸元が静かで過剰装飾なし
- ceremony ではなく photo shoot 用の整い方

State of Wear は sequence の進行とも強く関係する。

### 4. Forbidden Reinterpretation

衣装が再解釈されてはならない方向。

例：
- シンプルな花嫁衣装 → 豪華祭壇花嫁化しない
- ベールなし → 勝手にベールを足さない
- グローブなし → 勝手にグローブを足さない
- マーメイドライン → 別シルエットへ流れない
- 上品で静かな花嫁 → generic glamorous bride へ行かない

-----

## 基本原則

### 1. 衣装変更は人物再設計ではない

衣装を変えるとき、変わるのは服であって人物ではない。
Costume PAL は、衣装変更が別人化の口実にならないようにする。

### 2. 花嫁化は drift の危険工程である

ウェディングドレスは特に、人物同一性より役割類型を強く呼ぶ。
したがって花嫁化は一段ずつ進め、必要なら再アンカーを挟む。

### 3. 衣装の意味は sequence に従う

Costume PAL は単独で豪華さを競わない。
その衣装が sequence のどの役割を支えているかを優先する。

例：
- ブーケは起では保持、承で外してもよい
- 指輪は転以降で意味を持つ

衣装や小物の意味はコマ役割に従う。

### 4. 小物の継続は場面連続性に直結する

ブーケ、指輪、眼鏡のような要素は、
残っているか消えているかだけで
場面の連続性に大きく影響する。
小物の扱いは装飾ではなく、連続性の管理項目である。

### 5. Costume PAL は Character PAL より下位

衣装が人物を壊す場合は、衣装の正しさより人物同一性を優先する。
ドレスが少し揺れても、人物が保たれる方が重要である。

-----

## 他 PAL との関係

優先順位：

1. Character PAL
2. Sequence PAL
3. Costume PAL
4. Background PAL
5. Style PAL

Costume PAL は重要だが、
人物や sequence の意味を壊してまで守るべきものではない。
ただし Background PAL よりは上位である。
理由は、衣装と小物は人物の直近の連続性に関わるからである。

-----

## 起こしやすい Drift

- **Archetype Drift**：衣装が役割 archetype を強く呼び、
  人物がその archetype に飲まれること
  （花嫁 → generic bridal beauty など）
- **Formality Drift**：フォーマル化に伴って、
  人物が整いすぎた別類型へ流れること
- **Accessory Drift**：小物が勝手に追加・消失・変形すること
  （ベール追加、グローブ追加、ブーケ消失、
  指輪位置の誤り、左右逆転 など）
- **Silhouette Drift**：衣装のシルエットが別衣装へ流れること
  （マーメイドライン → 別の広がり方、
  オフショルダー → 別形状 など）
- **Cultural Drift**：衣装の文化的記号が
  人物の文化的類型を変えること
  （ウェディングドレス → 欧米花嫁 archetype へ吸着 など）

-----

## 典型例

### 日常服（ニュートラルアンカー）
- 白いシャツ
- フレアスカート
- 自然な立ち姿
- 感情的シチュエーションなし

人物をニュートラルに呼び出すための衣装アンカーになりやすい。

### 花嫁衣装
- 白いマーメイドライン
- ベールなし
- グローブなし
- 小さなイヤリング
- 細いチェーンネックレス
- ブーケあり / なしは状態差分で管理
- 指輪は sequence の意味に応じて段階的に強調

花嫁衣装は強いが drift を招きやすい。
Costume PAL と anti-drift を一体で運用する必要がある。

-----

## 意義

Costume PAL の意義は、衣装をその場の装飾ではなく、
人物・場面・意味を支える継続資産として扱えるようにすることにある。

これにより以下が可能になる：

- 同じ人物が着替えても同じ人に見える
- 小物の有無に意味を持たせられる
- sequence の中で衣装状態を遷移管理できる
- archetype drift を抑えながら役割表現ができる

Costume PAL は、衣装を人物を壊す要因から
人物を支える要因へ変えるための層である。
