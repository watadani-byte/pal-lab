# Background PAL

## 定義

Background PAL とは、同じ場面・同じ空間・同じ光を、
人物同一性やシーケンスの連続性を壊さずに維持するために、
空間の骨格・光・情報量・禁止再解釈を固定する継続層である。

Background PAL は、単なる背景指定ではない。
舞台装置の一覧でもない。
Background PAL は、背景を

- どこか
- どういう空間か
- どのような光か
- どの程度の情報量か
- 何を再解釈してはならないか

として固定し、人物の同一性やシーケンスの連続性を
支えるための記述層である。

-----

## 目的

Background PAL の目的は、美しい背景を毎回描かせることではない。
目的は、背景が変化しても以下の状態を維持することである。

- 同じ場所として読める
- 同じ場面の継続として読める
- 人物の類型 drift を誘発しない
- シーケンスの説得力を壊さない

Background PAL は、背景を豪華にするためではなく、
人物と時間の連続性を支えるために存在する。

-----

## なぜ Background PAL が必要か

背景は人物ほど目立たないが、複数枚を並べた時には非常に重要である。
背景が少し変わるだけで、別会場・別テイク・別シーン・
カット集に見えることがある。

また、背景はしばしば人物側にも影響を与える。

例：
- 欧風の空間が欧米人化を誘発する
- 豪華な会場が generic glamorous bride を呼ぶ
- 屋外挙式が archetype drift を強める

したがって、背景も人物と同様に同一存在として管理される必要がある。

-----

## Background PAL が固定するもの

### 1. Spatial Core

その背景をその背景たらしめる空間の骨格。

例：
- フォトウェディングの明るい室内スタジオ
- 白からアイボリーのニュートラル空間
- 中央に奥行きがある
- 左右に装花や立体配置がある
- 撮影用の清潔で静かな空間

Spatial Core は、背景の最小同一性である。

### 2. Light Signature

背景に固有の光の性質。

例：
- 柔らかい拡散光
- ハイキーだが白飛びしすぎない
- 影は弱く、人物をきつくしない
- 色温度は中立からやや暖かい
- dramatic light にはしない

Light Signature は、背景の空気感と人物の類型 drift を
左右するため、非常に重要である。

### 3. Scene Density

背景の情報量と演出密度。

例：
- 情報量は控えめ
- 人物を食わない
- 装花はあるが主役にならない
- 高級感はあっても豪華化しすぎない
- 撮影空間として静かで整理されている

Scene Density は、背景が archetype を
勝手に呼び込まないための制御でもある。

### 4. Forbidden Reinterpretation

背景が再解釈されてはならない方向。

例：
- スタジオ → 教会化しない
- フォトウェディング → 屋外挙式化しない
- 室内 → ガーデン ceremony 化しない
- 清潔な空間 → 豪華ホテル化しない
- シンプルな背景 → 過剰な祭壇化をしない

詳細は `pal/anti_drift.md` を参照。

## 基本原則

### 1. 1枚目で定義し、以後は固定命令に切り替える

背景は最初の基準画像で定義する。
2枚目以降は背景の内容を毎回説明せず、
以下のように固定命令へ切り替える：

- 同じ背景
- 同じスタジオ
- 同じ光
- 背景変更なし
- 背景再解釈なし

背景を毎回説明し直すと、
同じ概念の別背景に再生成されやすい。

### 2. 背景は人物を支えるが、人物より優先しない

背景の格や雰囲気が人物同一性を壊す場合、
Background PAL より Character PAL を優先する。

### 3. 背景は archetype を呼び込まない方がよい

特に以下のような背景は人物の drift を強めることがある：
- ガーデン挙式
- 教会
- 豪華会場
- 西洋 ceremony 空間

同一人物性を重視する場合は、
フォトウェディングのような人物主導の空間の方が安定しやすい。

### 4. 構図変更と背景維持は分けて考える

full body → waist-up → bust-up のような構図変化があっても、
背景そのものは同じであるべきである。
カメラ位置の変更を、背景変更の口実にしてはならない。

-----

## 他 PAL との関係

Background PAL は通常、
Character PAL・Sequence PAL・Costume PAL を支える従属層である。

優先順位：

1. Character PAL
2. Sequence PAL
3. Costume PAL
4. Background PAL
5. Style PAL

背景は重要だが、人物やシーケンスを壊してまで守るべきものではない。

-----

## 起こしやすい Drift

- **Venue Drift**：同じ背景のつもりが別会場になること
  （スタジオ → 教会、屋内 → 屋外 など）
- **Ceremony Drift**：背景が儀式性を強く呼び、
  人物まで ceremony archetype に吸われること
- **Style Drift**：背景の質感や装飾が強すぎて、
  人物がその様式へ巻き込まれること
- **Cultural Drift**：背景の文化的記号が
  人物の文化的類型を再解釈させること
  （欧風会場 → 欧米人化、豪華会場 → generic glamorous bride など）

## 典型例

### フォトウェディング・スタジオ（安定型）
- 明るい室内
- 柔らかい拡散光
- ニュートラルな白〜アイボリー
- 上品だが静か
- 撮影用空間
- ceremony を強制しない

### 屋外ガーデン挙式（注意型）
- 緑、花、通路、自然光
- 演出としては美しい
- ただし ceremony archetype と欧米花嫁類型を呼び込みやすい

同一人物性を優先する場合、
フォトウェディング・スタジオの方が
Background PAL として安定しやすい。

-----

## 意義

Background PAL の意義は、背景を単なる装飾やその場の雰囲気ではなく、
人物とシーケンスの継続性を支える構成要素として
運用できるようにすることにある。

これにより以下の読みが成立しやすくなる：

- 同じ人物が同じ場所にいる
- 同じ場面が続いている
- 構図だけが変わっている
- 時間だけが進んでいる

Background PAL は、複数枚の連続性を
人物以外の側から支える固定層である。
