# Image Generation Continuity Handoff

This file is an operational handoff for image-generation continuity work
within the PAL LAB continuity stack.

It is not a research document and not part of the repository-level governance framing.
It records the active generation stack, operational order, prompt conventions,
and evaluation principles for current work.

-----

## Framework Context

This repository is a PAL-based continuity framework for identity-consistent
image generation and related governance/control work.

Read the following files before beginning generation work:

1. `README.md`
1. `protocols/operational_protocol.md`
1. `protocols/prompt_protocol.md`
1. `notes/identity_consistency_evaluation.md`

-----

## Non-Negotiable Rules

Do not compress this framework into a generic “character consistency prompt system.”

Do not collapse Character, Costume, Background, Sequence, Protocol, and Evaluation
into a single layer.
Treat them as separate but connected control layers.

- Human judgment remains final.
- Continuity is not authority.
- Persistence is not trust.
- Stable reconstruction is not correctness.
- Working is not canonical.
- Drifted frames must not automatically become anchors.

-----

## Operational Order

For image-generation continuity work, follow this operational order:

1. Anchor / Character fixed
1. Costume changed
1. Small prop or accessory added
1. Background changed
1. Working reference fixed
1. Human identity check performed
1. Sequence started

-----

## Prompt Writing Conventions

- Separate design text from execution text.
- Keep execution prompts short.
- Use reference / keep / change only / do not structure.
- Prefer Japanese for design and evaluation.
- Prefer English for final execution prompts.

-----

## Evaluation Principles

- Use both perceptual consistency and structural consistency.
- Do not treat visual quality as proof of identity consistency.
- If drift appears, prefer stopping and reverting to the last accepted
  reference rather than forcing correction loops.

-----

## Mode Priority

When uncertain, prioritize:

1. Character Lock Mode
1. Balanced Mode
1. Quality-First Mode

If drift is visible, move back toward the stricter lock mode.

-----

## Acknowledgement

This is a layered continuity system, not a single prompt recipe.
Each layer — Character, Costume, Background, Sequence, Protocol,
Evaluation — has a distinct role and must be treated accordingly.
