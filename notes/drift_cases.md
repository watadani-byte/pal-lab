# Drift Cases

実際の生成で確認された drift を記録し、
再発防止に用いるためのメモである。
drift は偶然の失敗として扱わず、命名し、工程ごとに切り分ける。

*drift の分類と定義は `pal/anti_drift.md` を参照。*

-----

## 記録フォーマット

各 drift ケースは以下の形式で記録する。
観測（何を見たか）と判定（どう判断したか）を分離する。

```
### [ケース名]

**Observation（観測）:**
生成結果として何が確認されたかを客観的に記述する。
例：鼻筋が基準より強くなり、顔の重心が上に移動した。

**Judgment（判定）:**
それをどう判断したかと、採用 / 不採用の結論を記述する。
例：Cultural Drift と判定。不採用。PAL の禁止再解釈に該当する。

**工程:**
どの工程で発生したか。
例：花嫁化時

**再発防止:**
プロンプトまたは PAL 側への書き戻し内容。
例：anti_drift.md の禁止再解釈に鼻筋強化を明記する。
```

-----

## 代表例

- 欧米人化
- 花嫁 archetype 化
- ガーデン挙式への背景再解釈
- フォトリアル jump
- 三つ編み drift
- ベール / グローブの勝手な追加
- open cheerful smile への表情 drift
- full body 化に伴う理想化
- 横顔化に伴う骨格の再解釈

-----

## 工程別の危険点

- アンカー時
- 花嫁化時
- full body 化時
- 横顔化時
- 笑顔化時
- スタイル変更時

-----

## 運用

- drift が出た画像は採用しない
- 何が drift だったかを明示的に記録する
- 「綺麗に見える失敗」を特に疑う
- 同じ drift が繰り返される場合は、
  プロンプトではなく PAL 側へ書き戻す
